📋 Visão Geral

Sistema de logging profissional para Delphi, projetado com foco em performance, extensibilidade e robustez. Suporta múltiplos destinos de log com processamento assíncrono em background.

  • ✓ Processamento Assíncrono - Logs processados em thread separada sem bloquear a aplicação
  • ✓ Múltiplos Handlers - Arquivo, Email, Push Notification e extensível
  • ✓ Formato JSON - Logs estruturados para fácil análise e integração
  • ✓ Thread-Safe - Sincronização robusta para ambientes multi-thread
  • ✓ Zero Memory Leaks - Gerenciamento automático via interfaces
  • ✓ Rotação de Arquivos - Controle automático de tamanho dos logs
  • ✓ Filtro por Nível - Configure quais níveis cada handler processa
  • ✓ Clean Architecture - Código limpo seguindo SOLID e Design Patterns

🚀 Quick Start

Comece a usar o logger em minutos com estes exemplos práticos.

Uso Básico

A forma mais simples de começar a usar o logger:

Delphi 
program BasicLoggerDemo;

uses
  Logger.Types,
  Logger.Interfaces,
  Logger.Factory;

var
  Logger: ILogger;
begin
  // Cria logger com handler de arquivo padrão
  Logger := TLoggerFactory.CreateDefaultLogger;
  
  // Registra mensagens de diferentes níveis
  Logger.Debug('Iniciando processamento');
  Logger.Info('Usuário autenticado com sucesso');
  Logger.Warning('Cache próximo do limite');
  Logger.Error('Falha na conexão com banco de dados');
  
  // Aguarda processamento antes de encerrar
  Logger.Flush;
  Logger.Shutdown;
end.

Usando Singleton

Acesse a mesma instância do logger de qualquer lugar da aplicação:

Delphi 
unit MyBusinessUnit;

interface

procedure ProcessOrder(OrderId: Integer);

implementation

uses
  Logger.Facade;

procedure ProcessOrder(OrderId: Integer);
begin
  // Acessa singleton de qualquer lugar
  TLogger.Instance.Info(
    Format('Processando pedido #%d', [OrderId]),
    'Orders'  // Categoria
  );
  
  try
    // ... lógica de negócio ...
    TLogger.Instance.Debug('Pedido validado', 'Orders');
  except
    on E: Exception do
    begin
      TLogger.Instance.Error(
        Format('Erro no pedido #%d: %s', [OrderId, E.Message]),
        'Orders'
      );
      raise;
    end;
  end;
end;

end.
💡
Dica: O singleton é inicializado automaticamente no primeiro acesso. Lembre-se de chamar TLogger.ReleaseInstance ao finalizar a aplicação.

Múltiplos Handlers

Configure diferentes destinos para seus logs:

Delphi 
procedure ConfigureLogger;
var
  Logger: ILogger;
  FileHandler: ILogHandler;
  EmailHandler: ILogHandler;
  PushHandler: ILogHandler;
begin
  Logger := TLogger.Instance;
  
  // Handler de arquivo - todos os níveis
  FileHandler := TLoggerFactory.CreateFileHandler(
    'C:\Logs\myapp.log',
    ALL_LOG_LEVELS
  );
  Logger.RegisterHandler(FileHandler);
  
  // Handler de email - apenas Warning e Error
  EmailHandler := TLoggerFactory.CreateEmailHandler(
    'smtp.empresa.com',      // Host SMTP
    587,                        // Porta
    'sistema@empresa.com',     // Usuário
    'senha123',                // Senha
    'noreply@empresa.com',     // Remetente
    ['admin@empresa.com'],     // Destinatários
    True,                        // Usar SSL
    [llWarning, llError]         // Níveis habilitados
  );
  Logger.RegisterHandler(EmailHandler);
  
  // Handler de push - apenas Error
  PushHandler := TLoggerFactory.CreatePushHandler(
    'https://api.pushserver.com/send',
    'API_KEY_123',
    'MinhaApp',
    30000,
    [llError]
  );
  Logger.RegisterHandler(PushHandler);
end;

Logs com Dados Extras

Adicione informações contextuais aos seus logs:

Delphi 
procedure LogUserAction(UserId: Integer; const Action, IP: string);
var
  ExtraData: TJSONObject;
begin
  ExtraData := TJSONObject.Create;
  try
    // Adiciona dados contextuais
    ExtraData.AddPair('user_id', TJSONNumber.Create(UserId));
    ExtraData.AddPair('action', Action);
    ExtraData.AddPair('ip_address', IP);
    ExtraData.AddPair('user_agent', 'Mozilla/5.0...');
    ExtraData.AddPair('session_id', 'abc123xyz');
    
    // Log com método genérico
    TLogger.Instance.Log(
      llInformation,
      'Ação do usuário registrada',
      'UserActivity',
      ExtraData
    );
  finally
    ExtraData.Free; // Seguro: logger faz cópia interna
  end;
end;

📊 Níveis de Log

O sistema suporta quatro níveis de log, cada um com propósito específico.

Nível Badge Uso Recomendado Método
Debug DEBUG Informações detalhadas para diagnóstico durante desenvolvimento Logger.Debug()
Information INFO Eventos normais da aplicação (login, transações, etc.) Logger.Info()
Warning WARNING Situações anormais que não impedem funcionamento Logger.Warning()
Error ERROR Erros que afetam funcionalidade ou requerem atenção Logger.Error()

Filtrando por Nível

Configure quais níveis cada handler deve processar:

Delphi 
// Handler que processa todos os níveis
FileHandler := TLoggerFactory.CreateFileHandler('app.log', ALL_LOG_LEVELS);

// Handler apenas para Warning e Error
EmailHandler := TLoggerFactory.CreateEmailHandler(
  { ... config ... },
  [llWarning, llError]
);

// Handler apenas para Error
PushHandler.EnabledLevels := [llError];

// Alterar níveis em runtime
FileHandler.EnabledLevels := [llInformation, llWarning, llError]; // Remove Debug

📁 File Handler

O handler de arquivo é o destino padrão para logs. Suporta rotação automática de arquivos quando atingem um tamanho máximo configurável.

Configuração

Delphi 
var
  Handler: ILogHandler;
begin
  // Configuração básica - usa caminho padrão
  Handler := TLoggerFactory.CreateFileHandler;
  
  // Configuração completa
  Handler := TFileLogHandler.Create(
    'C:\Logs\application.log',  // Caminho do arquivo
    ALL_LOG_LEVELS,                 // Níveis habilitados
    10485760,                      // Tamanho máximo: 10 MB
    5                               // Máximo de arquivos backup
  );
  
  Logger.RegisterHandler(Handler);
end;
Parâmetro Tipo Padrão Descrição
AFilePath string Logs\application.log Caminho completo do arquivo de log
AEnabledLevels TLogLevels ALL_LOG_LEVELS Níveis de log a processar
AMaxFileSize Int64 10485760 (10 MB) Tamanho máximo antes de rotacionar
AMaxBackupFiles Integer 5 Quantidade de arquivos de backup

Rotação de Arquivos

📱 Push Notification Handler

Envia logs para servidores de push notification via HTTP POST. Compatível com Firebase Cloud Messaging, OneSignal, e outros serviços.

Configuração

Delphi 
var
  PushHandler: ILogHandler;
begin
  PushHandler := TLoggerFactory.CreatePushHandler(
    'https://fcm.googleapis.com/fcm/send',  // URL do servidor
    'YOUR_SERVER_KEY',                      // API Key
    'MinhaAplicacao',                       // Nome da app
    30000,                                   // Timeout em ms
    [llError]                                 // Apenas erros críticos
  );
  
  Logger.RegisterHandler(PushHandler);
end;

⚙️ Criando Handler Customizado

Implemente a interface ILogHandler ou herde de TBaseLogHandler para criar handlers personalizados.

Exemplo: Handler para Console

Delphi 
unit Logger.Handlers.Console;

interface

uses
  System.SysUtils,
  Logger.Types,
  Logger.Interfaces,
  Logger.Handlers.Base;

type
  TConsoleLogHandler = class(TBaseLogHandler)
  private
    FUseColors: Boolean;
    
    procedure SetConsoleColor(ALevel: TLogLevel);
    procedure ResetConsoleColor;
  protected
    procedure DoHandle(const AEntry: ILogEntry); override;
  public
    constructor Create(
      AUseColors: Boolean = True;
      AEnabledLevels: TLogLevels = ALL_LOG_LEVELS
    );
    
    property UseColors: Boolean read FUseColors write FUseColors;
  end;

implementation

uses
  Winapi.Windows;

constructor TConsoleLogHandler.Create(AUseColors: Boolean; AEnabledLevels: TLogLevels);
begin
  inherited Create('ConsoleHandler', AEnabledLevels);
  FUseColors := AUseColors;
end;

procedure TConsoleLogHandler.SetConsoleColor(ALevel: TLogLevel);
var
  LHandle: THandle;
  LColor: Word;
begin
  if not FUseColors then Exit;
  
  LHandle := GetStdHandle(STD_OUTPUT_HANDLE);
  
  case ALevel of
    llDebug:       LColor := FOREGROUND_INTENSITY;                              // Cinza
    llInformation: LColor := FOREGROUND_GREEN or FOREGROUND_INTENSITY;         // Verde
    llWarning:     LColor := FOREGROUND_RED or FOREGROUND_GREEN or FOREGROUND_INTENSITY; // Amarelo
    llError:       LColor := FOREGROUND_RED or FOREGROUND_INTENSITY;           // Vermelho
  else
    LColor := FOREGROUND_RED or FOREGROUND_GREEN or FOREGROUND_BLUE;
  end;
  
  SetConsoleTextAttribute(LHandle, LColor);
end;

procedure TConsoleLogHandler.ResetConsoleColor;
var
  LHandle: THandle;
begin
  if not FUseColors then Exit;
  
  LHandle := GetStdHandle(STD_OUTPUT_HANDLE);
  SetConsoleTextAttribute(LHandle, FOREGROUND_RED or FOREGROUND_GREEN or FOREGROUND_BLUE);
end;

procedure TConsoleLogHandler.DoHandle(const AEntry: ILogEntry);
var
  LOutput: string;
begin
  LOutput := Format('[%s] [%s] %s: %s', [
    FormatDateTime('yyyy-mm-dd hh:nn:ss.zzz', AEntry.Timestamp),
    LOG_LEVEL_NAMES[AEntry.Level],
    AEntry.Category,
    AEntry.Message
  ]);
  
  SetConsoleColor(AEntry.Level);
  try
    WriteLn(LOutput);
  finally
    ResetConsoleColor;
  end;
end;

end.

Exemplo: Handler para Database

Delphi 
unit Logger.Handlers.Database;

interface

uses
  System.SysUtils,
  Data.DB,
  FireDAC.Comp.Client,
  Logger.Types,
  Logger.Interfaces,
  Logger.Handlers.Base;

type
  TDatabaseLogHandler = class(TBaseLogHandler)
  private
    FConnection: TFDConnection;
    FTableName: string;
    FOwnsConnection: Boolean;
  protected
    procedure DoHandle(const AEntry: ILogEntry); override;
  public
    constructor Create(
      AConnection: TFDConnection;
      const ATableName: string = 'LOG_ENTRIES';
      AEnabledLevels: TLogLevels = ALL_LOG_LEVELS;
      AOwnsConnection: Boolean = False
    );
    destructor Destroy; override;
  end;

implementation

constructor TDatabaseLogHandler.Create(
  AConnection: TFDConnection;
  const ATableName: string;
  AEnabledLevels: TLogLevels;
  AOwnsConnection: Boolean);
begin
  inherited Create('DatabaseHandler', AEnabledLevels);
  FConnection := AConnection;
  FTableName := ATableName;
  FOwnsConnection := AOwnsConnection;
end;

destructor TDatabaseLogHandler.Destroy;
begin
  if FOwnsConnection and Assigned(FConnection) then
    FConnection.Free;
  inherited;
end;

procedure TDatabaseLogHandler.DoHandle(const AEntry: ILogEntry);
var
  LQuery: TFDQuery;
begin
  LQuery := TFDQuery.Create(nil);
  try
    LQuery.Connection := FConnection;
    LQuery.SQL.Text := Format(
      'INSERT INTO %s (TIMESTAMP, LEVEL, CATEGORY, MESSAGE, JSON_DATA) ' +
      'VALUES (:ts, :level, :cat, :msg, :json)',
      [FTableName]
    );
    
    LQuery.ParamByName('ts').AsDateTime := AEntry.Timestamp;
    LQuery.ParamByName('level').AsString := LOG_LEVEL_NAMES[AEntry.Level];
    LQuery.ParamByName('cat').AsString := AEntry.Category;
    LQuery.ParamByName('msg').AsString := AEntry.Message;
    LQuery.ParamByName('json').AsString := AEntry.ToJSONString;
    
    LQuery.ExecSQL;
  finally
    LQuery.Free;
  end;
end;

end.

🔧 Configuração Avançada

Opções avançadas de configuração para cenários específicos.

Configuração via Arquivo INI

Delphi 
unit Logger.Config;

interface

uses
  System.SysUtils,
  System.IniFiles,
  Logger.Types,
  Logger.Interfaces,
  Logger.Facade,
  Logger.Factory;

type
  TLoggerConfigurator = class
  public
    class procedure LoadFromIniFile(const AFileName: string);
  end;

implementation

class procedure TLoggerConfigurator.LoadFromIniFile(const AFileName: string);
var
  LIni: TIniFile;
  LLogger: ILogger;
  LFilePath: string;
  LMaxSize: Int64;
  LEnableEmail: Boolean;
begin
  LIni := TIniFile.Create(AFileName);
  try
    LLogger := TLogger.Instance;
    
    // Configuração do File Handler
    if LIni.ReadBool('FileHandler', 'Enabled', True) then
    begin
      LFilePath := LIni.ReadString('FileHandler', 'Path', '');
      LMaxSize := LIni.ReadInteger('FileHandler', 'MaxSizeMB', 10) * 1024 * 1024;
      
      LLogger.RegisterHandler(
        TFileLogHandler.Create(LFilePath, ALL_LOG_LEVELS, LMaxSize)
      );
    end;
    
    // Configuração do Email Handler
    if LIni.ReadBool('EmailHandler', 'Enabled', False) then
    begin
      LLogger.RegisterHandler(
        TLoggerFactory.CreateEmailHandler(
          LIni.ReadString('EmailHandler', 'SMTPHost', ''),
          LIni.ReadInteger('EmailHandler', 'SMTPPort', 587),
          LIni.ReadString('EmailHandler', 'Username', ''),
          LIni.ReadString('EmailHandler', 'Password', ''),
          LIni.ReadString('EmailHandler', 'FromAddress', ''),
          LIni.ReadString('EmailHandler', 'ToAddresses', '').Split([';']),
          LIni.ReadBool('EmailHandler', 'UseSSL', True),
          [llWarning, llError]
        )
      );
    end;
  finally
    LIni.Free;
  end;
end;

end.

🧠 Gerenciamento de Memória

O sistema utiliza interfaces e contagem de referência para evitar vazamentos de memória.

Práticas Implementadas

  • ✓ Interfaces com ARC - Todas as classes principais implementam interfaces, garantindo liberação automática
  • ✓ Cópia de JSON - TLogEntry faz cópia do ExtraData para evitar problemas de ownership
  • ✓ Try-Finally - Todos os recursos temporários são protegidos com try-finally
  • ✓ Graceful Shutdown - Worker thread processa fila pendente antes de finalizar

Exemplo de Uso Seguro

Delphi 
procedure SafeLogging;
var
  ExtraData: TJSONObject;
begin
  // JSON temporário - pode ser liberado após a chamada
  ExtraData := TJSONObject.Create;
  try
    ExtraData.AddPair('key', 'value');
    
    // TLogEntry faz cópia interna do JSON
    TLogger.Instance.Log(llInfo, 'Mensagem', 'Cat', ExtraData);
  finally
    ExtraData.Free; // Seguro: logger já copiou os dados
  end;
end;

procedure ApplicationShutdown;
begin
  // Libera singleton e aguarda processamento
  TLogger.ReleaseInstance;
  
  // Verifica vazamentos em modo Debug
  {$IFDEF DEBUG}
  ReportMemoryLeaksOnShutdown := True;
  {$ENDIF}
end;

📐 Design Patterns Utilizados

O sistema implementa diversos padrões de projeto para garantir código limpo, extensível e de fácil manutenção.

🎯 Singleton

TLogger.Instance

Garante uma única instância global do logger, acessível de qualquer parte da aplicação.

🏭 Factory

TLoggerFactory

Centraliza e simplifica a criação de handlers com configurações pré-definidas.

👁️ Observer

TLogDispatcher

Handlers se registram e são notificados automaticamente para cada log entry.

🎭 Strategy

ILogHandler

Diferentes handlers implementam a mesma interface, permitindo intercâmbio.

🎪 Facade

TLogger

Interface simplificada que esconde a complexidade do sistema interno.

📦 Producer-Consumer

TLogQueue + TLogWorkerThread

Desacopla produção de logs do processamento assíncrono.

🔍 Troubleshooting

Soluções para problemas comuns encontrados ao usar o sistema de logger.

⚠️
Logs não aparecem no arquivo
  • Verifique se chamou Logger.Flush antes de verificar o arquivo
  • Confirme que o diretório existe e a aplicação tem permissão de escrita
  • Verifique se o handler está registrado e o nível está habilitado
⚠️
Emails não estão sendo enviados
  • Verifique credenciais SMTP e configurações de porta/SSL
  • Para Gmail, use App Password em vez da senha normal
  • Confirme que o nível do log está habilitado no handler
  • Verifique se há firewall bloqueando a porta SMTP
⚠️
Memory leaks ao encerrar
  • Sempre chame TLogger.ReleaseInstance antes de encerrar
  • Se usar instância própria, chame Logger.Shutdown
  • Evite manter referências a handlers após shutdown
⚠️
Aplicação trava ao encerrar
  • O Flush tem timeout de 5 segundos por padrão
  • Handlers lentos (email, push) podem atrasar o shutdown
  • Considere usar níveis mais restritivos para handlers de rede

// Em modo DEBUG, erros de handlers são enviados para OutputDebugString
