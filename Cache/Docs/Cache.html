1. Visão Geral e Arquitetura

Este módulo fornece uma solução completa de cache distribuído para aplicações Delphi 12, utilizando Redis como backend principal. A arquitetura foi projetada com foco em desacoplamento, resiliência e escalabilidade.

Diagrama de Arquitetura

┌─────────────────────────────────────────────────────────┐ │ CONSUMIDORES │ │ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐ │ │ │ Horse API │ │ Serviço X │ │ Worker Y │ │ │ └──────┬───────┘ └──────┬───────┘ └──────┬───────┘ │ └─────────┼─────────────────┼─────────────────┼───────────┘ │ │ │ ▼ ▼ ▼ ┌─────────────────────────────────────────────────────────┐ │ CAMADA DE ABSTRAÇÃO │ │ ┌────────────────────────────────────────────────────┐ │ │ │ ICacheManager (Alto Nível) │ │ │ │ GetString / PutString / GetObject<T> / GetOrAdd │ │ │ └───────────────────────┬────────────────────────────┘ │ │ │ │ │ ┌───────────────────────▼────────────────────────────┐ │ │ │ ICacheStrategy (Strategy Pattern) │ │ │ │ Get / Put / Remove / Exists / Clear / Stats │ │ │ └───────────────────────┬────────────────────────────┘ │ │ │ │ │ ┌───────────────┐ ┌────▼──────────┐ ┌────────────┐ │ │ │ICacheSerializer│ │TRedisCache │ │ICacheMonitor│ │ │ │ (Genérico) │ │ Strategy │ │ (Métricas) │ │ │ └───────────────┘ └──┬────┬──────┘ └────────────┘ │ └─────────────────────────┼────┼──────────────────────────┘ │ │ ┌───────────┘ └──────────┐ ▼ ▼ ┌──────────────────┐ ┌────────────────────┐ │ Redis Server │ │ Memory Fallback │ │ (delphiredis) │ │ (TDictionary) │ └──────────────────┘ └────────────────────┘ Threads auxiliares: ┌──────────────────┐ ┌────────────────────┐ │ Watchdog Thread │ │ Monitor Collector │ │ (reconexão) │ │ (histórico) │ └──────────────────┘ └────────────────────┘

Princípios Fundamentais

  • Zero acoplamento com o componente Redis — consumidores dependem apenas de interfaces
  • Fallback automático — cache em memória assume quando Redis está indisponível
  • Thread-safe — todas as operações protegidas por TCriticalSection
  • Reconexão automática — Watchdog monitora e reconecta sem intervenção
  • Métricas em tempo real — hits, misses, erros, hit ratio, uptime

2. Padrões de Projeto Aplicados

Padrão Tipo Onde é aplicado Benefício
Strategy GoF — Comportamental ICacheStrategy / TRedisCacheStrategy Desacopla o backend de cache dos consumidores. Permite trocar Redis por Memcached, arquivos etc.
Factory Method GoF — Criacional TCacheFactory Centraliza criação de instâncias; facilita injeção de dependência e testes
Singleton GoF — Criacional TCacheFactory (instâncias default) Garante instância única global com acesso controlado
Observer GoF — Comportamental TCacheEventProc / SetOnEvent Notifica interessados sobre conexão, desconexão, erros e fallback
Middleware / Chain of Responsibility GoF — Comportamental TCacheMiddleware no Horse Intercepta requisições de forma transparente sem alterar handlers existentes
Composition over Inheritance SOLID (DIP/ISP) TCacheManager compõe Strategy + Serializer Flexibilidade máxima na combinação de comportamentos

Princípios SOLID

Princípio Aplicação
S — Single Responsibility Cada classe tem uma única responsabilidade: TCacheConfig (configuração), TCacheSerializer (serialização), TMemoryFallbackCache (fallback), TRedisCacheStrategy (comunicação Redis)
O — Open/Closed Nova strategy (ex.: Memcached) pode ser adicionada implementando ICacheStrategy sem modificar código existente
L — Liskov Substitution Qualquer implementação de ICacheStrategy é intercambiável onde a interface é esperada
I — Interface Segregation Interfaces separadas: ICacheStrategy (operações), ICacheSerializer (serialização), ICacheMonitor (métricas)
D — Dependency Inversion Todas as dependências são injetadas via interface. O TCacheManager não conhece TRedisCacheStrategy diretamente

4. Módulo 1 — Cache Redis

4.1 Configuração (TCacheConfig)

Classe que centraliza todos os parâmetros do cache, eliminando valores hardcoded:

PropriedadeTipoDefaultDescrição
Hoststring127.0.0.1Endereço do servidor Redis
PortInteger6379Porta do Redis
ConnectionTimeoutInteger3000Timeout de conexão (ms)
ReadTimeoutInteger3000Timeout de leitura (ms)
DefaultTTLSecondsInteger300TTL padrão em segundos
MaxRetriesInteger3Tentativas de conexão
RetryIntervalMsInteger1000Intervalo entre tentativas
WatchdogIntervalMsInteger10000Intervalo do watchdog
PurgeIntervalMsInteger60000Intervalo de self-purge
KeyPrefixstringapp:cache:Prefixo das chaves Redis
FallbackEnabledBooleanTrueAtiva/desativa fallback local

4.2 Strategy de Cache (ICacheStrategy)

A interface ICacheStrategy define o contrato de comunicação com qualquer backend de cache. Nenhum consumidor conhece a implementação concreta.

Padrão Strategy: Permite trocar o backend (Redis → Memcached → Arquivo) sem alterar nenhuma linha de código nos consumidores. Basta criar uma nova classe que implemente ICacheStrategy.

Operações disponíveis:

MétodoDescrição
Get(Key, out Value): BooleanBusca valor. Retorna True se encontrado
Put(Key, Value, TTL)Armazena com TTL opcional
Remove(Key): BooleanRemove chave específica
Exists(Key): BooleanVerifica existência
ClearLimpa todo o cache
GetStats: TCacheStatsRetorna estatísticas atuais

4.3 Fallback Automático

Quando o Redis está indisponível, o TRedisCacheStrategy ativa automaticamente o TMemoryFallbackCache: um cache local thread-safe baseado em TDictionary protegido por TCriticalSection.

Fluxo de leitura com fallback:

GET("minha_chave") │ ▼ ┌──────────────┐ SIM ┌─────────────────┐ │Redis online? ├───────────►│ Redis.GET(key) │ └──────┬───────┘ └────────┬────────┘ │ NÃO │ ▼ ┌───────▼────────┐ ┌──────────────────┐ │ Encontrou? │ │Fallback.GET(key) │ └───┬────────┬───┘ └──────┬───────────┘ SIM│ │NÃO │ ▼ ▼ ▼ Retorna Cache Miss Resultado Valor (incrementa miss) (incrementa fallback count)

4.4 Watchdog de Conexão

Uma thread dedicada executa periodicamente:

  1. Envia PING ao Redis para verificar saúde da conexão
  2. Se falhar, marca como desconectado e dispara evento ceDisconnected
  3. Em cada ciclo, tenta reconectar se estiver desconectado
  4. Ao reconectar, dispara evento ceReconnected
  5. Executa ciclo de self-purge conforme PurgeIntervalMs

4.5 Serialização Genérica (ICacheSerializer)

Utiliza REST.Json (nativo do Delphi) para converter objetos Delphi ↔ JSON string de forma transparente via RTTI.

// Armazenar objeto
var LPessoa := TPessoa.Create;
LPessoa.Nome := 'Rodrigo';
LPessoa.Idade := 30;
CacheManager.PutObject<TPessoa>('pessoa:1', LPessoa, 600);
LPessoa.Free;

// Recuperar objeto
var LRecuperada: TPessoa;
if CacheManager.GetObject<TPessoa>('pessoa:1', LRecuperada) then
begin
  ShowMessage(LRecuperada.Nome); // 'Rodrigo'
  LRecuperada.Free;
end;

4.6 GetOrAdd (Cache-Aside Pattern)

O método GetOrAdd<T> implementa o padrão Cache-Aside: tenta buscar do cache; se não encontrar, executa a factory fornecida, salva o resultado e retorna.

var LProduto := CacheManager.GetOrAdd<TProduto>('produto:42',
  function: TProduto
  begin
    Result := ProdutoRepository.FindById(42); // query ao banco
  end,
  300 // TTL 5 minutos
);
Atenção: O objeto retornado por GetOrAdd é de propriedade do chamador. Você é responsável por chamar Free quando não precisar mais dele.

5. Módulo 2 — Monitor de Cache

O TCacheMonitor implementa ICacheMonitor e oferece:

5.1 Métricas em Tempo Real

MétricaDescrição
TotalHitsTotal de acertos no cache
TotalMissesTotal de cache miss
TotalErrorsTotal de erros de comunicação
TotalFallbacksVezes que o fallback foi ativado
TotalEvictionsTotal de remoções explícitas
CurrentKeysChaves atuais (aproximado)
IsConnectedStatus da conexão Redis
HitRatioPercentual de acerto (hits / total)
UptimeSecondsTempo desde a inicialização

5.2 Histórico

O monitor coleta snapshots periódicos (configurável via StartCollecting), armazenando até AMaxHistory registros (padrão: 720, equivale a 1h com coleta a cada 5s).

// Inicia coleta a cada 5 segundos
TCacheFactory.Monitor.StartCollecting(5000);

// Recupera últimos 10 minutos de dados
var LHistory := TCacheFactory.Monitor.GetHistoricalStats(10);
try
  // LHistory é um TJSONArray com snapshots timestamped
  Writeln(LHistory.ToJSON);
finally
  LHistory.Free;
end;

5.3 Health Check

if TCacheFactory.Monitor.IsHealthy then
  // Redis está conectado e com menos de 100 erros
else
  // Atenção: possível problema de infraestrutura

6. Módulo 3 — Middleware Horse

6.1 Funcionamento

O TCacheMiddleware intercepta requisições GET no Horse:

Requisição GET /produtos?page=1 │ ▼ ┌─────────────────┐ │ Cache.Get(key) │ └────┬────────┬───┘ HIT│ │MISS ▼ ▼ ┌───────────┐ ┌──────────────┐ │ Retorna │ │ ANext() │ │ 200 + body│ │ (handler) │ │ PARA aqui │ └──────┬───────┘ └───────────┘ │ ▼ ┌───────────────┐ │ Captura body │ │ Cache.Put() │ │ Status 2xx? │ └───────────────┘

6.2 Geração de Chave

A chave do cache é gerada automaticamente usando MD5 do caminho + query string:

  • GET /produtos → horse:<md5 de "/produtos">
  • GET /produtos?page=1 → horse:<md5 de "/produtos?page=1">

6.3 Registro

// Forma 1: Usando New (recomendado)
THorse.Use(TCacheMiddleware.New(TCacheFactory.Manager, 120));

// Forma 2: Inicialização manual
TCacheMiddleware.Initialize(TCacheFactory.Manager, 120);
THorse.Use(TCacheMiddleware.Middleware);
Dica: Apenas respostas com status 2xx são armazenadas no cache. Erros (4xx, 5xx) nunca são cacheados.
Métodos não-GET (POST, PUT, DELETE) passam direto pelo middleware sem nenhuma interferência.

7. Módulo 4 — Testes Unitários e de Estresse

7.1 Testes Unitários (DUnitX)

Cobertura dos testes:

TCacheConfigTest

  • Valores padrão corretos
  • Validação de configuração válida e inválida
  • Clone independente (deep copy)
  • Montagem de chave com prefixo

TSerializerTest

  • Serialização de objeto para JSON string
  • Deserialização de JSON string para objeto
  • Roundtrip completo (serializa → deserializa → compara)
  • Exceção ao serializar nil
  • Exceção ao deserializar string vazia
  • Serialização/deserialização de TJSONValue
  • Exceção com JSON inválido

TFallbackCacheTest

  • Put e Get básicos
  • Get de chave inexistente
  • Remove e Exists
  • Clear completo
  • Expiração por TTL (com Sleep)
  • Evição quando MaxEntries é excedido

7.2 Testes de Estresse

Validam a robustez do TCriticalSection sob alta concorrência:

TesteThreadsOperações/ThreadObjetivo
Test_Concurrent_ReadWrite (10, 1000) 10 1.000 Carga leve — baseline
Test_Concurrent_ReadWrite (50, 5000) 50 5.000 Carga média — contenção moderada
Test_Concurrent_ReadWrite (100, 10000) 100 10.000 Carga alta — 1M de operações
Test_Concurrent_ReadWrite_NoDeadlock 20 2.000 (misto) Detecção de deadlock com timeout de 30s
Test_Concurrent_Mixed_Operations 30 500 (5 operações) Put/Get/Exists/Remove/Count simultâneos
Test_High_Volume_Sequential 1 200.000 Throughput sequencial (100k writes + 100k reads)

7.3 Executando os Testes

1. Abra o projeto Tests.Runner.dpr no Delphi 12
2. Compile (Ctrl+F9)
3. Execute (F9)
4. Os resultados aparecem no console
5. Relatório NUnit XML é gerado automaticamente
Nota: Os testes do Serializer e Fallback não dependem de Redis rodando. Apenas os testes de integração (se criados) precisariam de um Redis ativo.

8. Factory e Configuração

8.1 TCacheFactory

Ponto central de criação e acesso. Oferece dois modos:

Modo Singleton (padrão para aplicações)

// Na inicialização da aplicação
TCacheFactory.Initialize('10.0.0.5', 6379);

// Em qualquer lugar do código
var LManager := TCacheFactory.Manager;
LManager.PutString('chave', 'valor');

// Na finalização (opcional — feito automaticamente)
TCacheFactory.Finalize;

Modo Factory (para testes ou múltiplas instâncias)

var LConfig := TCacheConfig.Create;
LConfig.Host := 'redis-cluster-1';
LConfig.Port := 6380;

var LStrategy := TCacheFactory.CreateStrategy(LConfig);
var LSerializer := TCacheFactory.CreateSerializer;
var LManager := TCacheFactory.CreateManager(LStrategy, LSerializer);

LConfig.Free; // Config é clonada internamente

8.2 Eventos

TCacheFactory.Manager.SetOnEvent(
  procedure(AEvent: TCacheEvent; const AMessage: string)
  begin
    case AEvent of
      ceConnected:         Log('✅ ' + AMessage);
      ceDisconnected:      Log('❌ ' + AMessage);
      ceReconnected:       Log('🔄 ' + AMessage);
      ceError:             Log('⚠️ ' + AMessage);
      ceFallbackActivated: Log('🔀 ' + AMessage);
      cePurgeCompleted:    Log('🧹 ' + AMessage);
    end;
  end
);

9. Guia de Uso Completo

9.1 Setup Mínimo

uses
  Cache.Factory, Cache.Config;

// Inicializar
TCacheFactory.Initialize('127.0.0.1', 6379);

// Usar
TCacheFactory.Manager.PutString('hello', 'world', 60);

var LValue: string;
if TCacheFactory.Manager.GetString('hello', LValue) then
  Writeln(LValue); // 'world'

9.2 Com Horse (API REST)

uses
  Horse, Cache.Factory, Cache.Config, Cache.Middleware.Horse;

begin
  TCacheFactory.Initialize('127.0.0.1', 6379);
  
  // Registra middleware de cache (TTL = 2 minutos)
  THorse.Use(TCacheMiddleware.New(TCacheFactory.Manager, 120));
  
  THorse.Get('/dados',
    procedure(AReq: THorseRequest; ARes: THorseResponse)
    begin
      Sleep(2000); // Simula query demorada
      ARes.Send('{"resultado":"dados pesados"}')
          .ContentType('application/json');
    end
  );

  // Endpoint de monitoramento (sem cache)
  THorse.Get('/metrics',
    procedure(AReq: THorseRequest; ARes: THorseResponse)
    var LJSON: TJSONObject;
    begin
      LJSON := TCacheFactory.Monitor.GetStatsJSON;
      try
        ARes.Send(LJSON.ToJSON).ContentType('application/json');
      finally
        LJSON.Free;
      end;
    end
  );

  TCacheFactory.Monitor.StartCollecting;
  THorse.Listen(9000);
end.

9.3 Cache de Objetos Tipados

type
  TCliente = class
  private
    FId: Integer;
    FNome: string;
    FEmail: string;
  public
    property Id: Integer read FId write FId;
    property Nome: string read FNome write FNome;
    property Email: string read FEmail write FEmail;
  end;

// Salvar
var LCliente := TCliente.Create;
LCliente.Id := 1;
LCliente.Nome := 'Rodrigo';
LCliente.Email := 'rodrigo@email.com';
TCacheFactory.Manager.PutObject<TCliente>('cliente:1', LCliente, 600);
LCliente.Free;

// Recuperar
var LCached: TCliente;
if TCacheFactory.Manager.GetObject<TCliente>('cliente:1', LCached) then
begin
  Writeln(LCached.Nome); // 'Rodrigo'
  LCached.Free; // ← Não esquecer!
end;

// GetOrAdd (Cache-Aside)
var LProduto := TCacheFactory.Manager.GetOrAdd<TProduto>('produto:42',
  function: TProduto
  begin
    Result := FRepository.FindById(42);
  end,
  300
);
try
  Writeln(LProduto.Nome);
finally
  LProduto.Free;
end;
