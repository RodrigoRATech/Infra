RabbitQueue

Infraestrutura de filas de mensagens para Delphi 12 (Athens) sobre RabbitMQ via protocolo STOMP. Arquitetura orientada a interfaces, thread-safe, com handlers intercambiáveis e sistema de eventos completo.

Delphi 12 Athens RabbitMQ 3.5.1+ STOMP Protocol danieleteti/delphistompclient Thread-Safe SOLID Clean Architecture GoF Patterns
🏠

Visão Geral

O RabbitQueue é uma biblioteca Delphi 12 que encapsula toda a complexidade de comunicação com o RabbitMQ via protocolo STOMP, expondo uma API fluente, orientada a interfaces e totalmente thread-safe para produção e consumo de mensagens em background.

🎯
Objetivo Principal

Abstrair a infraestrutura de mensageria, permitindo que o código de negócio se concentre apenas na lógica dos handlers, sem conhecer detalhes de protocolo ou threading.

🔌
Dependência Externa

Utiliza exclusivamente a unit StompClient do repositório danieleteti/delphistompclient. Nenhuma outra dependência externa é necessária.

ℹ️

O transporte TCP é feito via Indy (padrão). Para usar Synapse, defina a diretiva de compilação USESYNAPSE no projeto.

RabbitQueue.Interfaces

Declara todos os contratos do sistema. É a única unit que deve ser referenciada pelo código de aplicação — nenhuma implementação concreta vaza para fora desta camada.

RabbitQueue.Handler.Base

Fornece duas implementações prontas para uso: uma base abstrata para herança e um handler anônimo para closures.

Classe abstrata — Template Method

RabbitQueue.Connection

Implementa IRabbitConnection adaptando a API do StompClient (danieleteti) para o contrato interno. É thread-safe via TCriticalSection e gerencia o ciclo de vida das subscriptions por fila.

⚠️

Subscribe por sessão: O método SubscribeIfNeeded garante que cada fila seja subscrita uma única vez por sessão de conexão, usando um TDictionary interno. Reconexões limpam o dicionário automaticamente.

Mapeamento de Prioridade

TQueuePriorityValor STOMP (header priority)
qpLow1
qpNormal5
qpHigh7
qpCritical9

Comportamento do Nack

ℹ️

O IStompClient do repositório danieleteti não possui método Nack nativo. A estratégia adotada é: para requeue (ARequeue=True), simplesmente não fazer ACK — o broker reentrega automaticamente após o timeout com amClient. Para descarte definitivo (ARequeue=False), faz ACK com header x-dead-letter: true.

🧵

RabbitQueue.Worker

Implementa o consumo de mensagens em background via TThread interna. Gerencia o ciclo de vida da thread, despacha mensagens para os handlers corretos e dispara todos os eventos do sistema.

🔒
Thread Safety

A lista de filas é copiada para um snapshot a cada ciclo via ToArray, liberando o lock durante operações de rede — evita contenção no acesso à fila de nomes.

📡
Sincronização de Eventos

Todos os callbacks de evento são disparados via TThread.Synchronize, garantindo execução na thread principal (VCL/FMX safe).

Ciclo do Worker Thread

🔄 Fluxo TWorkerThread.Execute
while not Terminated do Snapshot ← FQueueNames.ToArray // cópia thread-safe for each QueueName in Snapshot do Item ← Connection.Receive(QueueName, timeout=500ms) if Item.IsEmpty → FireOnQueueEmpty → continue Handler ← FindHandler(Item.Queue) if Handler = nil → Ack(Item) → continue FireOnBeforeProcess(Item) Result ← Handler.Execute(Item) FireOnAfterProcess(Item, Result) case Result of prSuccess → Ack(Item) prFailure → Nack(Item, Requeue=False) prRequeue → Nack(Item, Requeue=True) Sleep(10ms) // evita 100% CPU
🎯

RabbitQueue.Core

Implementa IRabbitQueue atuando como Facade sobre a conexão, o worker e os handlers. É a única classe com a qual o código de aplicação interage diretamente (via interface).

✅

Gerenciamento de memória: FWorker é declarado como IRabbitQueueWorker (interface), garantindo liberação automática por reference count. FHandlers e FQueueNames são objetos (TList) liberados explicitamente no destrutor.

Exemplo: Uso Básico

Configuração mínima para publicar e consumir mensagens de uma fila com um handler anônimo.

Pascal Uso básico — publicar e consumir
uses RabbitQueue.Factory, RabbitQueue.Interfaces, RabbitQueue.Types; var LConfig : TRabbitConnectionConfig; LQueue : IRabbitQueue; begin // 1. Configuração de conexão LConfig := TRabbitConnectionConfig.Default; LConfig.Host := 'localhost'; LConfig.Port := 61613; LConfig.Username := 'guest'; LConfig.Password := 'guest'; LConfig.MaxRetries := 3; // 2. Criação via factory com configuração fluente LQueue := TRabbitQueueFactory.Instance .CreateQueue(LConfig) .WithQueue('fila.pedidos') .WithPrefetch(1); // 3. Handler anônimo para processar pedidos LQueue.RegisterHandler( TRabbitQueueFactory.Instance.CreateHandler( 'PedidosHandler', 'fila.pedidos', function(AItem: TQueueItemInfo): TProcessResult begin // Sua lógica de negócio aqui Writeln('Processando pedido: ' + AItem.Body); Result := prSuccess; end ) ); // 4. Publicar uma mensagem LQueue.Enqueue('fila.pedidos', '{"id":1,"produto":"Notebook"}'); // 5. Iniciar consumo em background LQueue.Start; Writeln('Worker ativo. Pressione ENTER para parar...'); Readln; // 6. Parar o worker LQueue.Stop; end;

Exemplo: Handler Concreto via Herança

Para lógica complexa ou reutilizável, crie uma classe herdando de TRabbitQueueHandlerBase e implemente os hooks necessários.

Pascal Handler especializado para e-mail
uses RabbitQueue.Handler.Base, RabbitQueue.Types, System.SysUtils; type TEmailHandler = class(TRabbitQueueHandlerBase) strict protected /// Lógica principal — OBRIGATÓRIO implementar function DoExecute(const AItem: TQueueItemInfo): TProcessResult; override; /// Hook antes do processamento — OPCIONAL procedure DoBeforeExecute(const AItem: TQueueItemInfo); override; /// Hook após o processamento — OPCIONAL procedure DoAfterExecute(const AItem: TQueueItemInfo; const AResult: TProcessResult); override; /// Tratamento de erro interno — OPCIONAL function DoHandleError(const AItem: TQueueItemInfo; const AError: Exception): TProcessResult; override; end; function TEmailHandler.DoExecute( const AItem: TQueueItemInfo): TProcessResult; begin // Parse do JSON e envio do e-mail... Writeln('[EMAIL] Enviando para: ' + AItem.Body); Result := prSuccess; end; procedure TEmailHandler.DoBeforeExecute(const AItem: TQueueItemInfo); begin Writeln('[LOG] Iniciando: ' + AItem.MessageId); end; procedure TEmailHandler.DoAfterExecute(const AItem: TQueueItemInfo; const AResult: TProcessResult); begin Writeln('[LOG] Concluído. Resultado: ' + IntToStr(Ord(AResult))); end; function TEmailHandler.DoHandleError(const AItem: TQueueItemInfo; const AError: Exception): TProcessResult; begin Writeln('[ERRO] Falha no envio: ' + AError.Message); Result := prRequeue; // recoloca na fila para nova tentativa end; // ─── Registro na fila ─────────────────────────────────────────────────── LQueue.RegisterHandler( TEmailHandler.Create('EmailHandler', 'fila.email') );

Exemplo: Configuração de Eventos

O sistema de eventos é baseado em reference to procedure (closures). Todos os callbacks são executados na thread principal via TThread.Synchronize.

Pascal Todos os eventos disponíveis
// ── Eventos de fila ─────────────────────────────────────────────────── LQueue.SetOnEnqueue(procedure(const AItem: TQueueItemInfo) begin // Dispara ao publicar uma mensagem (Enqueue) Writeln('[+] Enfileirado em ' + AItem.Queue + ': ' + AItem.Body); end); LQueue.SetOnDequeue(procedure(const AItem: TQueueItemInfo) begin // Dispara após um item ser consumido com sucesso Writeln('[-] Desenfileirado: ' + AItem.MessageId); end); LQueue.SetOnQueueEmpty(procedure(const AQueueName: string) begin // Dispara quando a fila não tem mensagens no timeout Writeln('[~] Fila vazia: ' + AQueueName); end); LQueue.SetOnQueueError(procedure(const AQueueName: string; const AError: Exception) begin // Dispara em erros de comunicação ou processamento Writeln('[!] Erro em ' + AQueueName + ': ' + AError.Message); end); // ── Eventos de processamento ────────────────────────────────────────── LQueue.SetOnBeforeProcess(procedure(const AItem: TQueueItemInfo) begin // Dispara ANTES do handler processar o item Writeln('[>] Iniciando: ' + AItem.MessageId); end); LQueue.SetOnAfterProcess(procedure(const AItem: TQueueItemInfo; const AResult: TProcessResult) begin // Dispara APÓS o handler concluir com sucesso case AResult of prSuccess : Writeln('[✓] Processado: ' + AItem.MessageId); prFailure : Writeln('[✗] Descartado: ' + AItem.MessageId); prRequeue : Writeln('[↩] Reenfileirado: ' + AItem.MessageId); end; end); LQueue.SetOnProcessError(procedure(const AItem: TQueueItemInfo; const AError: Exception; var AShouldRequeue: Boolean) begin // Dispara quando o handler lança uma exceção // AShouldRequeue controla se a mensagem volta para a fila Writeln('[!!] Falha: ' + AError.Message); AShouldRequeue := AItem.RetryCount < 3; // até 3 tentativas end);

Exemplo: Múltiplas Filas com Prioridades

Exemplo completo de produção com múltiplas filas, handlers distintos e publicação com diferentes níveis de prioridade.

Pascal Cenário completo de produção
uses RabbitQueue.Factory, RabbitQueue.Interfaces, RabbitQueue.Types, RabbitQueue.Handler.Base; var LConfig : TRabbitConnectionConfig; LQueue : IRabbitQueue; LFactory : IRabbitQueueFactory; begin LFactory := TRabbitQueueFactory.Instance; // ── Conexão ──────────────────────────────────────────────────────────── LConfig := TRabbitConnectionConfig.Default; LConfig.Host := 'rabbit.meuservidor.com'; LConfig.Username := 'admin'; LConfig.Password := 's3cr3t'; LConfig.VirtualHost := 'producao'; LConfig.MaxRetries := 5; LConfig.RetryDelay := 2000; // ── Configuração fluente ─────────────────────────────────────────────── LQueue := LFactory.CreateQueue(LConfig) .WithQueue('fila.email') .WithQueue('fila.sms') .WithQueue('fila.push') .WithQueue('fila.auditoria') .WithPrefetch(3); // ── Handlers ─────────────────────────────────────────────────────────── LQueue .RegisterHandler(TEmailHandler.Create('EmailHandler', 'fila.email')) .RegisterHandler(LFactory.CreateHandler('SmsHandler', 'fila.sms', function(AItem: TQueueItemInfo): TProcessResult begin Writeln('[SMS] ' + AItem.Body); Result := prSuccess; end)) .RegisterHandler(LFactory.CreateHandler('PushHandler', 'fila.push', function(AItem: TQueueItemInfo): TProcessResult begin Writeln('[PUSH] ' + AItem.Body); Result := prSuccess; end)) .RegisterHandler(LFactory.CreateHandler('AuditoriaHandler', 'fila.auditoria', function(AItem: TQueueItemInfo): TProcessResult begin Writeln('[AUDIT] ' + AItem.Body); Result := prSuccess; end)); // ── Publicar com diferentes prioridades ─────────────────────────────── LQueue .Enqueue('fila.email', '{"to":"user@mail.com","subject":"Boas-vindas"}') .Enqueue('fila.sms', '{"phone":"+5562999999999","msg":"OTP: 4521"}', qpCritical) .Enqueue('fila.push', '{"device":"tok123","title":"Nova msg"}', qpNormal) .Enqueue('fila.auditoria','{"action":"LOGIN","user":"rodrigo"}', qpLow); // ── Eventos ──────────────────────────────────────────────────────────── LQueue.SetOnQueueError(procedure(const AQ: string; const E: Exception) begin Writeln('ERRO em ' + AQ + ': ' + E.Message); end); LQueue.SetOnProcessError(procedure(const AItem: TQueueItemInfo; const E: Exception; var ARequeue: Boolean) begin ARequeue := AItem.RetryCount < LConfig.MaxRetries; Writeln(Format('Tentativa %d/%d para %s', [AItem.RetryCount + 1, LConfig.MaxRetries, AItem.MessageId])); end); // ── Iniciar ──────────────────────────────────────────────────────────── LQueue.Start; Writeln('Sistema ativo. ENTER para encerrar...'); Readln; LQueue.Stop; end;

Referência: API Real do StompClient

A tabela abaixo documenta os métodos reais da IStompClient conforme o repositório danieleteti/delphistompclient (branch master, última atualização 2022-03-23).

🚫

Métodos que NÃO existem na interface real: SetServer, SetUser, SetHeartBeat, WaitForReceipt, Nack. Uso destes nomes causa erro de compilação E2003 Undeclared identifier.

Método RealAssinaturaObservação
NewStomp StompUtils.NewStomp: IStompClient Factory function. Não use TStompClient.Create diretamente.
Connect procedure Connect(Host; Port; Login; Passcode; VirtualHost) Todos os parâmetros de autenticação são passados aqui. Sem setters separados.
Disconnect procedure Disconnect Encerra a sessão STOMP e o socket TCP.
Send procedure Send(Destination; Body; Headers: IStompHeaders = nil) Destino deve ter prefixo /queue/ ou /topic/.
Subscribe procedure Subscribe(Destination; Ack: TAckMode; Headers: IStompHeaders) Deve ser chamado uma única vez por fila por sessão.
Receive function Receive(var Frame: IStompFrame; Timeout: Integer): Boolean Retorna True se recebeu frame no timeout. Frame é nil se retornar False.
Ack procedure Ack(MessageID: string) Apenas um parâmetro. Sem headers. Erro E2010 se passar IStompHeaders.
IsConnected function IsConnected: Boolean Verifica estado da conexão TCP.
NewHeaders StompUtils.NewHeaders: IStompHeaders Cria headers vazios. Encadeie com .Add(Key, Value).
